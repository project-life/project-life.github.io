CCSP 領域 5：實施雲端安全運營
雲端安全不只是建立防禦機制，還需要確保防禦系統能夠長期穩定運作。這正是 CCSP 領域 5 的核心所在——它聚焦於雲端安全運營，涵蓋如何維持雲端環境的安全，包括從強化基礎設施到事件管理的各種細節。

雖然許多操作安全原則與傳統 IT 相似，但雲端環境有其獨特的挑戰和機會。本領域將討論如何在雲端設計與管理實體和邏輯基礎架構，實施關鍵的運營控制，並管理安全運營。不論您是否處理虛擬化技術、網路安全或數位取證，領域 5 都能提供實用知識，幫助您確保雲端環境安全且高效運作。

5.1 建置與管理雲端基礎設施

硬體與虛擬環境的安全配置

  1. BIOS 與 UEFI：
    • 傳統系統使用 BIOS 來初始化硬體。現代系統則採用更安全的 UEFI，提供額外的安全層級。
    • 雲端供應商負責保護這些硬體層，以避免未授權存取。
  2. 虛擬化工具集：
    • 安裝在虛擬機上的監控工具，如 Azure Monitor Agent，可集中管理並快速檢查系統配置、健康狀態與更新狀況。
  3. 虛擬機器備份：
    • 備份虛擬機器的配置數據非常重要，因為它能在發生故障時快速重建系統。

5.2 營運與維護雲端基礎設施

本地與遠端存取控制

  1. 本地存取：
    • 使用控制台或 KVM（鍵盤、視訊顯示器與滑鼠）來直接管理伺服器。
  2. 遠端存取：
    • 使用 RDP 或 SSH 安全存取伺服器。VPN 是額外的安全選項，可加密遠端連線。
  3. 虛擬桌面：
    • 透過虛擬桌面基礎架構（VDI），使用者可從任何裝置安全地存取工作環境。

網路安全配置

  • 使用 TLS 加密資料傳輸。
  • DNSSEC 防止 DNS 攻擊，確保域名解析安全。
  • VPN 提供安全的遠端存取，IPsec 則為 VPN 提供加密支援。

防火牆與安全控制

  • 防火牆：過濾進出流量，防止未授權存取。
  • 入侵偵測與防禦系統（IDS/IPS）：即時檢測並阻止威脅。
  • 蜜罐與蜜網：誘捕攻擊者進一步行動，蒐集攻擊資訊以分析。

5.3 實施運營控制與標準

  1. 變更管理：
    • 管理系統變更，確保穩定性與安全性。
  2. 持續改善：
    • 定期檢查並提升 IT 服務的效能與效率。
  3. 事件管理：
    • 檢測並解決影響服務的問題，並防止類似事件再次發生。

5.4 支援數位取證

數位取證的核心概念

  • 收集、保存與分析電子證據，確保證據完整性。
  • 關鍵方法包括建立監管鏈（Chain of Custody）以記錄證據處理過程。

雲端取證的挑戰與解決方案

  • 雲端提供者應協助提供虛擬磁碟或機器的快照，而非存取實體硬體，避免侵犯其他租戶的隱私。

5.5 管理與相關方的溝通

有效的溝通對象包括：

  • 雲端供應商、客戶、合作夥伴、監管機構以及其他利害關係人。

5.6 安全操作管理

安全營運中心（SOC）

  • SOC 全天候運作，負責監控、分析與回應安全事件。

日誌管理與分析

  • 使用 SIEM 系統集中日誌，進行分析以偵測威脅。
  • 確保日誌時間一致性，使用 NTP 同步系統時鐘。

CCSP 考試準備小提示

  • 徹底掌握雲端安全運營的實務技能。
  • 對基礎設施、事件管理與取證方法有全局性的理解。
